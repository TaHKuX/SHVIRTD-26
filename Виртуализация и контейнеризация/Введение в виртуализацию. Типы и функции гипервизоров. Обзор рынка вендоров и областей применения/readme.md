### Задача 2
- Высоконагруженная база данных MySql, критичная к отказу  
Ответ: **физические сервера**, позволяет по максимуму использовать ресурсы сервера
- Различные web-приложения  
Ответ: **виртуализация уровня ОС**, каждое web-приложение будет жить в отдельном независимом контейнере, проще обновлять и откатывать обновления
- Windows-системы для использования бухгалтерским отделом  
Ответ: **паравиртуализация**, экономия ресурсов + разный доступ к ВМ с разными бухгалтерскими приложениями
- Системы, выполняющие высокопроизводительные расчёты на GPU  
Ответ: **физические сервера**, прямой доступ к максимуму ресурсов GPU, без всякий прослоек
### Задача 3
1. 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.  
Ответ: **VMware vSphere**, сбалансированный + универсальный + лидер рынка
2. Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.  
Ответ: **Xen**, надёжный + стабильный + универсальный + с открытым кодом, но лучше подойдёт бесплатный **Hyper-V Server**
3. Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.  
Ответ: **Xen**, надёжный + стабильный + универсальный + с открытым кодом
5. Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.  
Ответ: **KVM**, лучшее решение с открытым кодом для Linux
### Задача 4
Проблемы / недостатки: 
- Сложность обслуживания (к каждой платформе нужен свой подход)
- Требуется обширный уровень знаний у обслуживающего персонала (по каждой платформе)
- Для каждой платформы могут требоваться индивидуальные решения (например для бекапов)
- Неоптимальное распределение ресурсов
- Невозможность управлять всем из 1-го места

Для минимизации рисков / проблем:
- Стандартизация и унификация
- Использование единого портала для обслуживания
- Стремление к переходу на меньшее количество платформ
